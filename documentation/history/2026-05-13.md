## Source code

- `src/main.ts`: removed the `node-fetch` import and the four `globalThis.fetch/Headers/Request/Response` overrides that addressed the SDK's CORS limitations. Replaced by an Obsidian-native fetch adapter (see below).
- `src/app/utils/obsidian-fetch.ts` (new): `fetch`-shaped wrapper around Obsidian's `requestUrl`. Supports the subset of the Fetch API that the Replicate SDK uses on the `replicate.run()` code path: string/URL/Request input, method, headers, string/ArrayBuffer body, plus a standard `Response` for the return value. Throws on unexpected body types rather than coercing them.
- `src/app/utils/get-replicate-client.fn.ts`: passes `fetch: obsidianFetch` to `new Replicate(...)` so SDK requests now go through Obsidian's request layer.
- `src/app/utils/log.ts`: suppressed all `console.*` calls in the shipped bundle (kept the function shape behind comments).
- `src/app/plugin.ts`: added `-- reason` to the `obsidianmd/commands/no-plugin-id-in-command-id` directive; converted the async `addCommand` callback and the async `onClick` editor-menu handler to non-async functions that fire `void this.generateImages(...)` so they no longer trip `no-misused-promises`.
- `src/app/utils/generate-images.fn.ts`: extracted a `formatOutputItem(unknown): string` helper that handles strings, `{ url: string }` objects, and falls back to `JSON.stringify` â€” avoids the `no-base-to-string` directive and the `restrict-template-expressions` warnings (object/unknown). Error template literal narrowed via `error instanceof Error ? error.message : JSON.stringify(error)`.
- `eslint.config.ts`: migrated from `tseslint.config(...)` to `defineConfig([...])` (from `eslint/config`) so the deprecation warning clears.
- `package.json`: removed `node-fetch` from dependencies; bumped `packageManager` to `bun@1.3.14`.
